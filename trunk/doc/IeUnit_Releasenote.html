Release Notes

Changes in version 2.3.240 (Dec 9, 2006)

  1. Upgraded the system to .Net 2.0 platform. 
  2. Open sourced the QuickFocus tool that is implemented in C# 2.0.

Changes in version 2.1.238 (May  26, 2006)

  1. Extended Win32Dom to allow fetch text property. Added test case GetUrl.jst.

Changes in version 2.1.237 (May  17, 2006)

  1. Changed the default value for _ieWindowVisible from "false" to "true" (thanks to Stephen Newton).

Changes in version 2.1.236 (May  8, 2006)

  1. Extended file type configuration for ".jst" and ".sbk" to enable extra script command-line arguments.

Changes in version 2.1.234 (May  2, 2006)

  1. Removed obsolate file IeGuiRunner.html and IeGuiRunner.wsf.
  2. Added default break points for debugging mode.
  3. Added an entry to the Explorer's Send To menu. We can now execute a selected set of jst script by sending them to the IeUnit entry.

Changes in version 2.1.225(April 12, 2006)

  1. Added the method startProgram() and spanSkbProcess() to simplify handling of modal dialogs.

Changes in version 2.1.221 (Oct 22, 2005)

  1. Changed waitForModalDialog: it now uses this.findTimeout as timeout value to find the modal dialog window. this.win and this.comWin are not initialized with this.doc.parentWidnow instead of this.doc.window, the formmer allows provides more access the the modal dialog window.
  2. Added methods findTextArea() and setTextArea() methods to support TextArea elements (code contributed by Andy Rosenbaum).


Changes in version 2.1.220 (Sept 30, 2005)

  1. Fixed a defect relating to the -v option for hidding IE window.
  2. Extended Win32Dom to enable accessing modal dialog window through DHTM model. See ModalDialogTest.jst for more details.
  3. Deprecated the IeGuiRunner.wsf. Removed all related documentation and test code, the test runner IeGuiRunner.wsf itself will be supported till the next major release.

Changes in version 2.1.217 (June 16, 2005)

  1. Added the global variable _ to replace the variable _currentCase. The variable _currentCae remain defined to maintain compatibility with older versions.
  2. Fixed problem findByTagAndAttr(): can't search for  'class' attributes.

Changes in version 2.1.215 (June 9, 2005)

  1. Added _TestRunInit() to allow invidual configuration for each repeated test runs.
  2. Fixed a defect in findByTagAndAttr(): some attribute are not detected.

Changes in version 2.1.211 (May 31, 2005)

  1. Fixed yet another unicode problem.
  2. The SmartBookmark.init, if exisits, is now included directly before the code in .sbk file.  Previously, it was loaded outsides of the SmartBookmarkTest class.  With this change we can assimilate additonal classes for sbk scripts in SmartBookmark.init.

Changes in version 2.1  (May 21, 2005)

  1. Set the global variable _currentCase ( or simply _) for smartbook scripts, so that we can directly access the IeDhtml  methods in any generic JavaScript functions.
  2. Fixed an unicode related defect in QuickFocus: it now handles properly characters in the range 128-255. Thanks Kiril Jovchev for reporting this problem.
  3. Varius bug fixes and enhancements in QuickFocus  (NullReferences, IFrames, fonts, property details for focused elements, etc.).
  4. Extended Win32Dom to support mouse controls. The test MouseTest.jst demonstrates how to use these new features to drive modal dialogs.
  5. The class Win32Dom is now assimilated by the IeUnit class. Thus, normal test cases that assimilates IeUnit no long need to assimilate Win32Dom. Notice: this change might cause name conflict  if  existing scripts have used  method names which are used by Win32Dom.
  6. Updated IeUnit FAQ page. It contains now more information about handling of various kinds of popup windows
  7. Added global function RequireVersion() to allow a script to require particular version (or higher) of IeUnit.


Major changes in version 2.0.0 (April 13, 2005)

  1. Added index parameter to setField()
  2. Added index parameter to clickButton(), findButton(), clickLink(), findLink().
  3. Renamed IeRun.wsf to SmarkBookmark.wsf.
  4. Made findLink(), clickLink() polymorphic. The substring can also be a regular expression in the format like /goto here/i.
  5. Split setBox() into setCheckBox() and checkRadioButton().
  6. Removed hasSelect(). Renamed assertSelectHas() to  assertSelectHasOption().  Renamed setSelect() to  setSelectOption(). Added fireEvent() and checkSubmit() call into setSelectOption().
  7. Added polymorphic to findObjById(), findField(), findRadionButton(), findSelect(). These methods now accept both index and id as parameter.
  8. The smart-bookmark script now access the IeUnit framework through the 'this' instead of the 'theApp' variable.
  9. Renamed assertHasText to assertPageHasText().
  10. Added findTableCell(idOrIdx, rowIdx, columIdx), where idOrIdx can be either the table index or table id.
  11. Added the global variable _currentCase to allow stand-alone function to access the currently executing test case. 
  12. The property this.win now points to this.doc.parentWindow that corresponds to the 'window'  object in in-browser JavaScript enviornment. The old this.win is now called this.comWin.
  13. Added the command-line option -runfiles to run a list of .jst files. The file name no long has to match the test case name. IeTextRunner scans the .jst file to figure out the test case name. IeTextRunner now only loads selected .jst files. This changes makes it easier to hold multiple variations of the same test cases in the same directory. The option -run no long accept file name as parameter.
  14. Smart bookmark script can now be encoded in unicode. Added test case UnicodeTest.jst to demo the usage. Notice that test case scripts has already been able to handle unicode scripts..
  15. Added the script DebugTest.bat to start IeUnit test script in debugger (a script debugger must be installed).  Extended Windows file types for .jst and .sbk we can launch the debugger by right-mous clicking on script files in the windows explorer.
  16. The IeUnit installation program now installs Win32Dom.dll that enables users to access to normal desktop windows. Several test cases  in samples\Win32DomTest directory demostrate the usage of  Win32Dom.dll.  Because of  the Win32Dom.dll module IeUnit now requires the Microsoft .NET 1.1 runtime enviornment which can be downloaded free of change from Microsoft web site.
  17. Added QuickFocus as a third-party GUI tool that helps inspect HTML pages and create IeUnit scripts. QuickFocus is donated by VisuMap Technologies Inc.
  18. IeUnit no long sets the first frame as default frame when a page has frames or iframes. The test script has to use setFrame(idxOrId) to set the current frame explicitly.  If the frame structure is specified by multiple nested framed documents, the argument idxOrId must be of the syntax "idxOrId1/idxOrId2/...". If the frame structure is specified by only one top level framed document, then a single  index or id can address any nested frame.
  19. Added clickImage() method.
  20. Added the %IEUNIT_HOME%/local as a place holder for custom libraries which will be searched for *.jst and *.js files by the default configuration.
  21. Added  assertTextContains() and assertTagHasText().
