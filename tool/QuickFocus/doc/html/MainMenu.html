QuickFocus Main Menu

The main menu provides menu driven user interface to most features of the application. Whereas most features can be accessed more quickly through another means like shortcut keys, the menu provides an easy way to explorer the main features of QuickFocus. The menu items are explained as follows:

File > Open File...

Load a HTML file from the local file system. You can also load a web link file (*.url) through this item.

File > Load Script...

Load an IeUnit script from the local file system into the code snippet window.

File > Save Script...

Save the code in the snippet window into the script file where the script code is originally loaded.

File > Save Script As...

Save the code in the snippet window into a new script file.

File > Exit

Close the current QuickFocus session. Most configuration settings like operation mode, last opened web page, etc. will be saved into the application configuration file QuickFocus.xml located in the same directory where QuickFocus.exe is located.

Favorites

Provide quick access to the favorites sites configured for Internet Explorer.

Tools > Logger Window...

Open the application logger window which logs various inspection and code snippets depending on current application settings. You can open and close the logger window at any time.

Tools > Toggle Mode

Toggle the operation mode of the application between attached and detached mode. In attached mode, you can inspect the loaded web page. In detached mode, you can use the QuickFocus like a web browser to navigate to any interested web pages.

Tools > Debug Snippet...

Launch a JavaScript debugger to debug the code in the snippet window. In order to use this feature a script debugger must be installed. The Microsoft Script Debugger can, for instance,  be downloaded for free. If you want the debugger to stop at a particular line in the code snippet window you can insert the statement "debugger;" at that line, then let the debugger run. This menu item offers a convenient way to debug any IeUnit test case or smart bookmark.

Tools > Options...

This menu item provides interface to change the following application options:

        
 Max History Length:
The maximum history length in the URL poll-down list.

Focus Frame Width:

The line width of the focus and select frame in the page window.
Max Key Text Length:
The maximum length of key text. When generating assert-statements QuickFocus normally only extract characteristics word up to length specified by this option.

Clear snippet window automatically:

If this flag is checked QuickFocus will automatically clear the code snippet window before generating new code. If this flag is not checked, QuickFocus will insert new code at the position of the current cursor in the snippet window.
Comment out alternative statements:

QuickFocus often generates multiple suggestions to validate or control the selected element. If this flag is checked, all alternative code lines will have comment prefix "//".  You can choose the alternative suggestion by removing the comment prefix.

Run QuickFocus in simulation mode:

If this flag is checked QuickFocus automatically executes the script in the code snippet window when you select an element in the page window. The simulation mode should only be used with "Clear snippet window automatically" flag unchecked.
