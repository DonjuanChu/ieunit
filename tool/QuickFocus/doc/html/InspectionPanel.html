The inspection panel

The inspection panel provides user interface to generate, edit and execute IeUnit scripts. The inspection panel has the following components:

The code snippet window

QuickFocus issues code snippet into this window. You can execute the code in this window by clicking on the "Run Snippet" button in the inspection panel. You can execute part of the code by marking part of the code snippet then clicking on the "Run Snippet" button. When you manually changed the content in this window you can undo the changes by pressing the Ctrl-Z key. This code snippet window has a context menu that allows quick editing and execution of selected code snippet.

Assertion checkbox

If this checkbox is checked QuickFocus will generate assertion based validating code snippet when ever an element in the page window gets selected.

Action checkbox

If this checkbox is checked QuickFocus will generate action code snippet (e.g. this.clickButton()) for the selected element when ever an element in the page window gets selected.

Alternative checkbox

When this checkbox is checked QuickFocus will generate alternative suggestions to validate or control the selected element.

SBK Submit button

Generate smart bookmark code snippet using the currently selected element  (which is normally a button element) as submit button. QuickFocus will generate a statement for each input element on the page. You usually have to adjust input values for the input elements directly in the snippet window. If  no element has been selected currently this task will only generate no-submitting statements (e.g. no clickButton statements). You can then click on a submit button to let QuickFocus to complete the SBK submit script.

Submit button

Generate submit code snippet. This button works similar as the SBK submit button except that this button doesn't issue the first this.openWindow()statement. When executing the code snippet via the "Run Snippet" button QuickFocus won't open a new Internet Explorer window, but execute the code directly against the page currently loaded in the page window.

Test case button

Generate an IeUnit test case template for the currently loaded URL. You usually need to insert more statements into the code snippet to perform real testing tasks. You can use the "Submit" button to insert testing code into the template.

Run Snippet button

Execute the code snippet in the snippet window. If you have marked some code segment in the snippet window QuickFoucs will only execute the marked code. Most code snippets start with a this.openWindow() statement, so that an new Internet Explorer window will be opened when executing those code snippet. If the code or the marked code doesn't contain a this.openWindow() statement QuickFocus will insert a this.seekAndSetWindow() statement, so that the code will be executed against the page currently loaded into the page window.

The test result will be reported through status line or separate message window.

Clear Snippet button

Clear the code in the snippet window.

Snippet dragging handle

You can generate a copy of the code snippet currently display in the snippet window by dragging this handle to the desktop. If the code snippet contains a JavaScript function definition line (as required by IeUnit test case format) a file with the extension ".jst" will be generated. Otherwise, QuickFocus will generate smart bookmark file with the extension ".sbk".
